    Project Idea

    Soil and air temperatures are a few of the factors that dictate the rate at which plants grow and consume nutrients (reference). My idea is to build on existing Soil Temperature Sensor technologies to enable users in the Adelaide Hills Region to make informed decisions about what to plant in specific soil conditions. Arduino and similar micro bit board hardware have been used previously in soil testing scenarios; using Arduino compatible software to create programs to manipulate this data. I intend to enhance this for users by coupling it with a user-friendly program that gives additional advice, matching the conditions with ideal crops.

    Gardening is part and parcel of living in the Adelaide Hills, picturesque agriculture and wine region. This advice is invaluable to gardeners of all skill levels and sure to contribute to their success.

    SOIL TEMPERATURE SENSOR & WHAT TO PLANT

    Simply put, the purpose of a soil temperature sensor is to measure and record the temperature of the soil it is subjected to. Coupled with an Arduino Uno board, users can maintain records of a specific area over time and apply this knowledge when choosing crops come planting time. The focus of my improvement is the use of Arduino compatible software to create programs and present data in a specific way. Arduino hardware is easily accessible, and assembly of components for soil temperature sensors is a simple process.

    A simplified example of this is described by users of the SparkFun Micro Bit Board in a “Micro: Climate Experiment Guide” (micro:climate Kit Experiment Guide - learn.sparkfun.com, 2020). The experiment uses an LED array to show the current soil temperature and when prompted, it simply says “Plant” or “Don’t Plant” based on parameters in the coding. I intend to enhance this by creating a computer program that stores records and timestamps in a database and when prompted, returns a list of crops that are compatible with these planting conditions. Programming will require parameters determined by research of historical planting conditions in the region.

    In terms of a minimum viable product, a basic version will function by recording input data and assign general information for a temperature range based on parameters set in coding. By keeping the function simple, the performance of the coding can be tested before delving into versions that assign more specific information. Based on previous examples, the other components for this hardware make it easily modifiable, and later developments will consider the durability of these components.

    The Arduino will control the data input and output but there are other components to consider. The temperature sensor component will be inserted into the soil, when prompted it will take a reading of the temperature, notifying users if the reading was not recorded properly. The program will assign this data to a temperature range and locate the output data which is relevant to this range. Users may require data for extended periods, and the Arduino itself offers a limited amount of space. Through a comparison of Arduino projects, connecting a USB to a computer is the favoured solution for storage, particularly because it provides a timestamp for data.

    The challenge will be creating a program that can interpret and manipulate the data received. Arduino offers its own Software known as IDE which can be used across numerous operating systems. Creating code in the Arduino programming language and using general data from the region will provide the foundation for the program, users will have the capacity to add their data and be able to personalise their reports.
